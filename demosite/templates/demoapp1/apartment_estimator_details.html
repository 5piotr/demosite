The Estimator

This is an estimator designed to predict apartment prices on the Polish real estate market. It is outputting 2 prices. One comes from random forest model, the other from artificial neural network. Below you can find information about its creation and evaluation.

Web scraping

Data required for training of the models have been acquired via web scraping from a Polish auction portal. Automatic data acquisition is not prohibited by portals terms of use. The procedure did not interfere with it's functioning. The result was a set of 100k unique observations with 14 parameters:

  • city - city in which the apartment is located
  • district - city district
  • voivodeship - Polish administration region
  • localization_y - geographic location: latitude
  • localization_x - geographic location: longitude
  • market - primary market, aftermarket
  • offer_type - sell (safety check)
  • area - total area of the apartment
  • rooms - total number of rooms in the apartment
  • floor - floor on which the apartment is located
  • floors - total number of floors in the building
  • build_yr - year in which the apartment was built
  • price - price of the apartment in PLN
  • url - address of the website with the advertisement

    • Data preparation

    Some observations had to be discarded. They were missing significant information that are highly impacting the final price. 18% of data had no price to begin with. The remaining 30% had no built year and 5% ware missing the information about the floors. Rest of the data ware fairly complete, only 1-3% values per variable ware missing. After the drop, 50k observations remained.
    The categorical data have been checked for consistency. Only information about the floors needed processing. In border cases they were descriptive instead of numerical.
    Information about the price per m2 have been added for further preprocessing and analysis.
    Data below 1st percentile and above 99th percentile for variables: "area", "price", "price_of_sqm" have been considered as outliers. The same was for "build_yr" lower than 1900. Some outliers have also been found in latitude and longitude. Points that they were describing were outside the Polish border despite clearly Polish administration information. In some rare cases, values of the coordinates were numbers with no decimal places, which suggested some kind of algorithmic assignment not in line with the actual location. After dropping the outliers, 45k observations remained.
    Skew and kurtosis values for the variables have been checked. No further processing ware needed.

    K-means clustering

    The best practice is to train the model on geolocation data organized into bins. Otherwise, the model will struggle to find patterns in a two-dimensional continuous plane with many local extremes. Final price is driven by the coordinate values, but not in a proportional manner.
    There are two ways to approach this issue. The first one is to use city/district values. They are naturally organizing the data into bins. However, it is getting problematic to assign new points to those bins. It requires address analysis and validation. Also, it is very likely that the model is not training on all possible locations. It will not be able to predict output for locations that it hasn't seen before.
    The second way is to use latitude and longitude values to create virtual bins. Covering the area by grid with predefined resolution will make it easy to group the existing data points and also to put any given point into an already existing bin.

    This method is simple but unfortunately not efficient. It is visible after zooming the map.

    625 bins grid is not enough to reflect the complexity of apartment price structure on the city level. There are also bins that are completely empty, with no points to train the model. To correctly group the data, at least x10 higher resolution is required.

    x10 denser grid means that there are now 62.5k bins with only a small amount of them covered by data points. This is clearly not a correct approach.

    There is a better way, the k-means clustering algorithm. It is grouping the locations into a predefined number of bins-clusters. The cluster size is correlated to point density. More dense regions have smaller clusters and less dense bigger ones. Clustering performed for this project has been done only on latitude and longitude values, so it is purely geometric. Despite the simplicity, it is doing a great job. Here is the clustering visualization for the whole country.

    There are no empty clusters. Each one has different size and shape. In the areas between cities where there aren't many apartments on sale, the clusters are big. They are getting smaller the closer to cities they are, and even finer in the city centers. This behavior is beneficial to price estimator because the apartment prices are changing quickly on small distances in the cities and are stable on bigger areas between them. The city clusters are more or less reflecting the district structure, or at least they are in the same size range.

    The cluster amount selection is purely arbitrary and was determined by experiment. Different values have been tested and 600 seems to be a good balance between accuracy and efficiency. Below you can see how the prices per m2 (represented by dot size) are correlating with the clusters (represented by dot color).

    Cluster parameter has been added to the data set and redundant location information (address and geolocation) removed. All parameters besides area and build year were treated as categories and required creation of dummy variables. After all transformations, the final set has 49k data points and 670 features.

    Random forest regression

    The number of estimators for random forest regressor have been selected by grid search algorithm. Tested values are between 10 and 100. With increasing number of estimators the model performance improvement was small so finally selected amount was 30. The small number of estimators is improving model size and estimation time.
    The R2 score for this model is 0.81 as established by 5-fold cross validation. Below you can see a comparison between the test data and the predictions and also error distribution.



    Artificial neural network estimator

    The data set has been scaled before feeding into the ANN. The model is composed of the following layers:

    1st hidden layer
    32 neurons, LeakyReLU activation, α=0.01
    2nd hidden layer
    16 neurons, LeakyReLU activation, α=0.01
    3rd hidden layer
    8 neurons, LeakyReLU activation, α=0.01
    4th hidden layer
    4 neurons, LeakyReLU activation, α=0.01
    Output layer
    1 neuron

    Training has been performed over 1000 epochs and batch size of 256. The R2 score for this model is 0.84. Below you can see the loss plot, a comparison between the test data and the predictions and error distribution.


    The data points are a bit more centered in comparison to the results of the random forest model.


    Models comparison and evaluation

    In order to compare the models, a special data frame has been prepared. Its data points are covering the whole country and besides the location, every other parameter is the same. They are describing the same, most common type of apartment. This frame was fed through both models, and absolute differences between each estimation have been saved and transformed into a heat map.

    It is safe to say that this absolute difference is a good indicator of estimation reliability. When it is close to zero, this means that both models managed to find the same regularities/dependencies and make good estimations. The probability that those regularities/dependencies are correct is high. On the other hand, when the difference is big, the models are struggling, and it is more sure that they are further from the truth.
    Some clusters are showing big differences between both estimations. The reason for that is visible after updating the map with the points on which the models ware trained.

    Better results are visible for clusters with the most points, in the cities and in suburban areas where the apartment trade is taking place. In those clusters the models had a lot of points to train on. Model accuracy is proportional to the amount of data used during training.
    The models are showing high accuracy in the regions that this estimator will most likely be used.
